# General

- Never create features unless explicitly requested.
- Never rename folders.
- Never rename files without asking.
- Never move files without asking.
- Never delete code without asking.
- Never install packages unless requested.

# React

- JavaScript only.
- Functional components only.
- Never use class components.
- Never use inline styles.
- Always use Tailwind CSS.
- Components must stay under 200 lines.
- Keep components focused on one responsibility.

# UI

- Minimalist.
- Clean.
- No gradients.
- No glassmorphism.
- White background.
- Neutral gray colors.
- Blue accent color.
- Plenty of spacing.
- Rounded corners.
- Readable typography.

# Architecture

- Reuse components.
- Avoid duplicated code.
- Keep folder structure consistent.
- Do not introduce abstractions unless necessary.

# Development

If a request is ambiguous,
ask before generating code.

Never guess.

Always explain your changes before applying them.
